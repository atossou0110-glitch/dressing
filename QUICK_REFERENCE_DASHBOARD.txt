💻 CODE EXAMPLES

1. LOG AN ACTION (Automatic via middleware)
───────────────────────────────────────────
  $product->update(['name' => 'New Name']);
  // Automatically logs: user_id, action, model_type, model_id, ip, user_agent

2. MANUAL LOGGING
─────────────────
  AuditLog::log(
    action: 'custom_action',
    modelType: 'Product',
    modelId: 1,
    description: 'Custom description'
  );

3. QUERY AUDIT LOGS
───────────────────
  AuditLog::where('action', 'update')->get();
  AuditLog::where('model_type', 'Product')->latest()->first();

4. EXPORT DATA
──────────────
  $export = app(ExportService::class);
  return $export->exportProductsCsv();

5. GET CHART DATA
─────────────────
  $analytics = app(DashboardAnalyticsService::class);
  $trends = $analytics->orderTrendData(30);
  $topProducts = $analytics->topProductsData(10);

📊 DATABASE

New Table: audit_logs
├─ id (BIGINT PK)
├─ user_id (FK to users)
├─ action (VARCHAR - create/update/delete)
├─ model_type (VARCHAR - Product/Review/etc)
├─ model_id (BIGINT - nullable)
├─ old_values (JSON - nullable)
├─ new_values (JSON - nullable)
├─ description (TEXT - nullable)
├─ ip_address (IP)
├─ user_agent (TEXT)
└─ created_at (TIMESTAMP)

Indexes:
├─ (user_id, created_at)
├─ (model_type, model_id)
└─ (action)

🐛 TROUBLESHOOTING

Q: Tests failing?
A: Run php artisan migrate --force first

Q: Exports not downloading?
A: Check admin middleware is applied to routes

Q: Audit logs empty?
A: Middleware needs to be registered in Kernel.php

Q: Slow dashboard?
A: Check database indexes are created

Q: Charts not showing?
A: Install Chart.js: npm install chart.js
